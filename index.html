How dose JavaScript run ?

Ans:JavaScript is an interpreted language. we do not have to compile the js code before sending it to the browser. JavaScript is a single-threaded language. The code run line by line. Running a js program from the command line you have to install Nodejs. VS Code will run this code automatically with the terminal. we use web browsers as a platform for executing the JavaScript code. Google chrome has a V8 engine. This executes the JavaScript code in the browser. Other browsers have their engines like SpiderMonkey, Chakra, Hermes engine.

Some things happened when running JavaScript 1: Parser 2: AST(Abstract Syntax Tree) 3: Conversion to Machine code 4: Machine code 5: The output of the code.Frist JavaScript engine Loads js Code. Then interpreter starts the application. The compiler gets the code. The compiler starts optimization and compilation. The incrementally optimizes the code. Then we see the output in a browser..JavaScript is single-threaded but it can do multiple things with the help of assistants.

difference between local storage and session storage ?

Ans: localStorage and sessionStorage both extend Storage. sessionStorage and localStorage are similar but the difference is the data in localStorage doesn't clear, data in sessionStorage is clear when the page is cleared. When a document is loaded in a tab in the browser, a unique page session gets created and assigned to the tab. The Getting data, Setting data, and Deleting data are the same process for both localStorage and sessionStorage. If the tab is cleared sessionStorage data is cleared automatically. But the data of local storage will not be cleared until the user clear it. the localStorage help to store the important data of the users.
